# CRM 合同自动创建

## 合同类型映射

| 用户说的 | 脚本文件 |
|---------|---------|
| 单笔 / 单笔合同 | 值得买单笔.py |
| 年框 / 年框合同 | 值得买年框.py |
| 赠版 / 赠版合同 / 增版 | 资源增版.py |

## 脚本目录

`/Users/smzdm/Desktop/erp-spider-more/Brms/newcrm/`

## 执行流程

### 1. 解析用户指令

从用户消息中提取：
- **合同类型**（必须）：单笔 / 年框 / 赠版，映射到对应脚本
- **创建数量**（可选，默认 1）：用户说"创建5个"则执行5次

若用户未明确类型，主动询问。

### 2. 检查 Chrome 远程调试服务

执行以下命令检查 9222 端口是否已被占用：

```bash
lsof -i :9222
```

- **若无输出（未启动）**：启动 Chrome 远程调试：

```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 --user-data-dir=/Users/smzdm/chrome &
```

等待 3 秒后再次检查端口确认启动成功。若启动失败，告知用户并停止。

- **若有输出（已启动）**：跳过，直接进入下一步。

### 3. 循环执行脚本

对于需要创建 N 个合同的情况，循环 N 次：

```
第 i 次（i = 1 到 N）：
  1. 执行脚本：/Users/smzdm/virtualenv/Plag_Ui/bin/python3 /Users/smzdm/Desktop/erp-spider-more/Brms/newcrm/[脚本文件]
  2. 等待脚本执行完成，检查退出码
  3. 若退出码 != 0 或输出包含错误信息：
     - 立即停止后续执行
     - 终止流程，在最终汇报中说明失败位置和原因
  4. 若还有下一个（i < N）：等待 5 秒后继续
  5. 过程中不要逐个汇报，全部执行完后只汇报一次最终结果
```

### 4. 完成汇报

全部执行完成后，汇总告知：
- 合同类型
- 成功创建数量
- 若有失败，说明失败位置和原因

## 注意事项

- 脚本执行超时设置为 120 秒（单个脚本），超时视为失败
- 出错立即停止，不继续后续创建
- 所有英文错误信息翻译成中文展示
- Chrome 远程调试启动是后台进程，不要等待它退出
